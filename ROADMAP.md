## üìã Project Overview

**Project Name:** Litoral Agency Publisher  
**Type:** Chrome Extension (Manifest V3) / Firefox Add-on  
**Purpose:** Publishes approved campaigns to social media platforms from the user's own browser.

### Core Functionality
- The extension acts as a **thin client** that polls the Litoral API for approved campaigns.
- Campaigns are claimed from a queue and scheduled on social platforms by directly manipulating each platform's native web UI via content scripts.
- Supported platforms: **Facebook**, **Instagram**, **TikTok** (stub), **Google Business Profile** (stub).

### Tech Stack
| Technology | Purpose |

| Tailwind CSS 3.4.17 | Styling |

### Key Architectural Patterns
1. **Monorepo** ‚Äî Separate packages for shared logic, storage, UI, and per-page entry points.
2. **Dynamic Content Script Registration** ‚Äî Content scripts are registered via `chrome.scripting.registerContentScripts()` at runtime, avoiding a `host_permissions` manifest allowlist.
3. **Message Passing** ‚Äî Extensive use of `chrome.runtime.sendMessage` / `chrome.tabs.sendMessage` for cross-context communication.
4. **JSend API Standard** ‚Äî API responses wrapped in `{ status: 'success', data: T }` or `{ status: 'error', message: string }`.
5. **Rate Limiting & Safety** ‚Äî Max 2 campaigns per 90-second cycle, alarms prevent overlapping polls.
6. **Error Resilience** ‚Äî 401 handling clears tokens, 6 consecutive failures trigger a system notification.

### Deployment Pipeline (Auto)
On every push to `main`:

1. **Build** ‚Äî `pnpm build` (Chrome) + `pnpm build:firefox`
2. **Test** ‚Äî `pnpm test:unit` + `pnpm test:e2e`
3. **Version** ‚Äî Bump `manifest.json` version via `pnpm update-version`
4. **Package** ‚Äî `pnpm zip` + `pnpm zip:firefox`
5. **Deploy** ‚Äî Upload to Chrome Web Store + Firefox Add-ons (using stored API keys)
6. **Tag** ‚Äî Git tag `v{version}` and release notes

## ‚úÖ Platform Integration Checklist

Use this checklist when adding or polishing a platform integration.

- [ ] **Login Detection** ‚Äî Can the content script detect if the user is logged into the platform?
- [ ] **Navigation** ‚Äî Can the script navigate to the post creation / scheduling page?
- [ ] **Media Upload** ‚Äî Can the script upload images/videos to the platform's composer?
- [ ] **Caption/Text** ‚Äî Can the script fill in the post caption or description?
- [ ] **Date/Time** ‚Äî Can the script set the scheduled date and time?
- [ ] **Schedule/Submit** ‚Äî Can the script click the final "Schedule" or "Post" button?
- [ ] **Success Detection** ‚Äî Can the script detect a successful post (URL change, confirmation message)?
- [ ] **Error Handling** ‚Äî Does the script report failures (DOM not found, upload error, login expired) to the background?
- [ ] **Rate Limiting** ‚Äî Does the script respect the global 90-second delay between platforms?
- [ ] **Cleanup** ‚Äî Does the script clean up any injected UI or event listeners?
